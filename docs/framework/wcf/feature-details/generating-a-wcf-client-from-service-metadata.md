# <a name="generating-a-wcf-client-from-service-metadata"></a><span data-ttu-id="421a4-102">根据服务元数据生成 WCF 客户端</span><span class="sxs-lookup"><span data-stu-id="421a4-102">Generating a WCF Client from Service Metadata</span></span>

 <span data-ttu-id="421a4-104">元数据文档可以位于持久性存储区中，也可以联机检索。</span><span class="sxs-lookup"><span data-stu-id="421a4-104">Metadata documents can be on a durable storage or be retrieved online.</span></span> <span data-ttu-id="421a4-105">联机检索遵循 WS-MetadataExchange 协议或 Microsoft 发现 (DISCO) 协议。</span><span class="sxs-lookup"><span data-stu-id="421a4-105">Online retrieval follows either the WS-MetadataExchange protocol or the Microsoft Discovery (DISCO) protocol.</span></span> <span data-ttu-id="421a4-106">Svcutil.exe 同时发出以下元数据请求以检索元数据：</span><span class="sxs-lookup"><span data-stu-id="421a4-106">Svcutil.exe issues the following metadata requests simultaneously to retrieve metadata:</span></span>  
  
- <span data-ttu-id="421a4-107">发往所提供地址的 WS-MetadataExchange (MEX) 请求。</span><span class="sxs-lookup"><span data-stu-id="421a4-107">WS-MetadataExchange (MEX) request to the supplied address.</span></span>  
  
- <span data-ttu-id="421a4-108">发往所提供地址的 MEX 请求，并追加了 `/mex`。</span><span class="sxs-lookup"><span data-stu-id="421a4-108">MEX request to the supplied address with `/mex` appended.</span></span>

 <span data-ttu-id="421a4-110">Svcutil.exe 基于 Web 服务描述语言 (WSDL) 或从服务接收的策略文件生成客户端。</span><span class="sxs-lookup"><span data-stu-id="421a4-110">Svcutil.exe generates the client based on the Web Services Description Language (WSDL) or policy file received from the service.</span></span> <span data-ttu-id="421a4-111">用户主体名称（UPN）是通过将用户名与 "" 连接起来生成的 \@ ，然后添加完全限定的域名（FQDN）。</span><span class="sxs-lookup"><span data-stu-id="421a4-111">The user principal name (UPN) is generated by concatenating the user name with "\@" and then adding a fully-qualified domain name (FQDN).</span></span> <span data-ttu-id="421a4-112">但是，对于在 Active Directory 上注册的用户，此格式无效，该工具生成的 UPN 导致 Kerberos 身份验证失败，并出现以下错误消息：**登录尝试失败。**</span><span class="sxs-lookup"><span data-stu-id="421a4-112">However, for users who registered on Active Directory, this format is not valid and the UPN that the tool generates causes a failure in the Kerberos authentication with the following error message: **The logon attempt failed.**</span></span> <span data-ttu-id="421a4-113">若要解决此问题，请手动修复该工具生成的客户端文件。</span><span class="sxs-lookup"><span data-stu-id="421a4-113">To resolve this problem, manually fix the client file that the tool generated.</span></span>  
  
```console

```
