# <a name="discards---c-guide"></a><span data-ttu-id="a3f26-103">弃元 - C# 指南</span><span class="sxs-lookup"><span data-stu-id="a3f26-103">Discards - C# Guide</span></span>

<span data-ttu-id="a3f26-104">从 C# 7.0 开始，C# 支持弃元，这是一种在应用程序代码中人为取消使用的临时虚拟变量。</span><span class="sxs-lookup"><span data-stu-id="a3f26-104">Starting with C# 7.0, C# supports discards, which are temporary, dummy variables that are intentionally unused in application code.</span></span> <span data-ttu-id="a3f26-105">弃元相当于未赋值的变量；它们没有值。</span><span class="sxs-lookup"><span data-stu-id="a3f26-105">Discards are equivalent to unassigned variables; they do not have a value.</span></span> <span data-ttu-id="a3f26-106">因为只有一个弃元变量，甚至不为该变量分配存储空间，所以弃元可减少内存分配。</span><span class="sxs-lookup"><span data-stu-id="a3f26-106">Because there is only a single discard variable, and that variable may not even be allocated storage, discards can reduce memory allocations.</span></span> <span data-ttu-id="a3f26-107">因为它们使代码的意图清楚，增强了其可读性和可维护性。</span><span class="sxs-lookup"><span data-stu-id="a3f26-107">Because they make the intent of your code clear, they enhance its readability and maintainability.</span></span>

<span data-ttu-id="a3f26-108">通过将下划线 (`_`) 赋给一个变量作为其变量名，指示该变量为一个占位符变量。</span><span class="sxs-lookup"><span data-stu-id="a3f26-108">You indicate that a variable is a discard by assigning it the underscore (`_`) as its name.</span></span> <span data-ttu-id="a3f26-109">例如，下面的方法调用返回 3 元组，其中的第一个和第二个值是弃元，*area* 是一个以前声明的变量，它将设置为 *GetCityInformation* 返回的相应的第三个部分:</span><span class="sxs-lookup"><span data-stu-id="a3f26-109">For example, the following method call returns a 3-tuple in which the first and second values are discards and *area* is a previously declared variable to be set to the corresponding third component returned by *GetCityInformation*:</span></span>

```csharp
(_, _, area) = city.GetCityInformation(cityName);
```

<span data-ttu-id="a3f26-110">在 C# 7.0 中，支持在以下上下文的分配中使用弃元：</span><span class="sxs-lookup"><span data-stu-id="a3f26-110">In C# 7.0, discards are supported in assignments in the following contexts:</span></span>

- <span data-ttu-id="a3f26-111">元组和对象[析构](deconstruct.md)。</span><span class="sxs-lookup"><span data-stu-id="a3f26-111">Tuple and object [deconstruction](deconstruct.md).</span></span>
- <span data-ttu-id="a3f26-112">使用 [is](language-reference/keywords/is.md) 和 [switch](language-reference/keywords/switch.md) 的模式匹配。</span><span class="sxs-lookup"><span data-stu-id="a3f26-112">Pattern matching with [is](language-reference/keywords/is.md) and [switch](language-reference/keywords/switch.md).</span></span>
- <span data-ttu-id="a3f26-113">对具有 `out` 参数的方法的调用。</span><span class="sxs-lookup"><span data-stu-id="a3f26-113">Calls to methods with `out` parameters.</span></span>
- <span data-ttu-id="a3f26-114">当范围内没有 `_` 时，独立的 `_`。</span><span class="sxs-lookup"><span data-stu-id="a3f26-114">A standalone `_` when no `_` is in scope.</span></span>

<span data-ttu-id="a3f26-115">当 `_` 是有效占位符时，尝试检索其值或在赋值操作中使用它时会生成编译器错误 CS0301：当前上下文中不存在名称 "\_"。</span><span class="sxs-lookup"><span data-stu-id="a3f26-115">When `_` is a valid discard, attempting to retrieve its value or use it in an assignment operation generates compiler error CS0301, "The name '\_' does not exist in the current context".</span></span> <span data-ttu-id="a3f26-116">这是因为 `_` 未赋值，甚至可能未分配存储位置。</span><span class="sxs-lookup"><span data-stu-id="a3f26-116">This is because `_` is not assigned a value, and may not even be assigned a storage location.</span></span> <span data-ttu-id="a3f26-117">如果它是一个实际变量，则不能像之前的示例那样对多个值使用占位符。</span><span class="sxs-lookup"><span data-stu-id="a3f26-117">If it were an actual variable, you could not discard more than one value, as the previous example did.</span></span>
