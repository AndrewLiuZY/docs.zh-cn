<span data-ttu-id="f16ac-105">.NET 实现包括可加快和优化开发过程并提供对系统功能的访问的类、接口、委托和值类型。</span><span class="sxs-lookup"><span data-stu-id="f16ac-105">.NET implementations include classes, interfaces, delegates, and value types that expedite and optimize the development process and provide access to system functionality.</span></span> <span data-ttu-id="f16ac-106">为了便于语言之间进行交互操作，大多数 .NET 类型都符合 CLS，因而可在编译器符合公共语言规范 (CLS) 的任何编程语言中使用。</span><span class="sxs-lookup"><span data-stu-id="f16ac-106">To facilitate interoperability between languages, most .NET types are CLS-compliant and can therefore be used from any programming language whose compiler conforms to the common language specification (CLS).</span></span>

- <span data-ttu-id="f16ac-113">调用 .NET Framework 安全检查。</span><span class="sxs-lookup"><span data-stu-id="f16ac-113">Invoke .NET Framework security checks.</span></span>

## <a name="naming-conventions"></a><span data-ttu-id="f16ac-118">命名约定</span><span class="sxs-lookup"><span data-stu-id="f16ac-118">Naming conventions</span></span>

 <span data-ttu-id="f16ac-119">.NET 类型使用点语法命名方案，该方案隐含了层次结构的意思。</span><span class="sxs-lookup"><span data-stu-id="f16ac-119">.NET types use a dot syntax naming scheme that connotes a hierarchy.</span></span> <span data-ttu-id="f16ac-120">此技术将相关类型分为不同的命名空间组，以便可以更容易地搜索和引用它们。</span><span class="sxs-lookup"><span data-stu-id="f16ac-120">This technique groups related types into namespaces so they can be searched and referenced more easily.</span></span> <span data-ttu-id="f16ac-121">全名的第一部分（最右边的点之前的内容）是命名空间名。</span><span class="sxs-lookup"><span data-stu-id="f16ac-121">The first part of the full name — up to the rightmost dot — is the namespace name.</span></span> <span data-ttu-id="f16ac-122">全名的最后一部分是类型名。</span><span class="sxs-lookup"><span data-stu-id="f16ac-122">The last part of the name is the type name.</span></span> <span data-ttu-id="f16ac-123">例如，`System.Collections.Generic.List<T>` 表示 `List<T>` 类型，属于 `System.Collections.Generic` 命名空间。</span><span class="sxs-lookup"><span data-stu-id="f16ac-123">For example, `System.Collections.Generic.List<T>` represents the `List<T>` type, which belongs to the `System.Collections.Generic` namespace.</span></span> <span data-ttu-id="f16ac-124">可使用 <xref:System.Collections.Generic> 中的类型使用泛型集合。</span><span class="sxs-lookup"><span data-stu-id="f16ac-124">The types in <xref:System.Collections.Generic> can be used to work with generic collections.</span></span>  
  
 <span data-ttu-id="f16ac-125">此命名方案使扩展 .NET Framework 的库开发人员可以轻松创建分层类型组，并用一致的、带有提示性的方式对其进行命名。</span><span class="sxs-lookup"><span data-stu-id="f16ac-125">This naming scheme makes it easy for library developers extending the .NET Framework to create hierarchical groups of types and name them in a consistent, informative manner.</span></span> <span data-ttu-id="f16ac-126">它还允许用全名（即命名空间和类型名称）明确地标识类型，这样可以防止类型名称发生冲突。</span><span class="sxs-lookup"><span data-stu-id="f16ac-126">It also allows types to be unambiguously identified by their full name (that is, by their namespace and type name), which prevents type name collisions.</span></span> <span data-ttu-id="f16ac-127">库开发人员在创建其命名空间的名称时应使用以下约定：</span><span class="sxs-lookup"><span data-stu-id="f16ac-127">Library developers are expected to use the following convention when creating names for their namespaces:</span></span>  
  
 <span data-ttu-id="f16ac-128">CompanyName.TechnologyName </span><span class="sxs-lookup"><span data-stu-id="f16ac-128">*CompanyName*.*TechnologyName*</span></span>  
  
 <span data-ttu-id="f16ac-129">例如，`Microsoft.Word` 命名空间就符合此原则。</span><span class="sxs-lookup"><span data-stu-id="f16ac-129">For example, the namespace `Microsoft.Word` conforms to this guideline.</span></span>  
  
 <span data-ttu-id="f16ac-130">利用命名模式将相关类型分组为命名空间是生成和记录类库的一种非常有用的方式。</span><span class="sxs-lookup"><span data-stu-id="f16ac-130">The use of naming patterns to group related types into namespaces is a very useful way to build and document class libraries.</span></span> <span data-ttu-id="f16ac-131">但是，此命名方案对可见性、成员访问、继承、安全性或绑定无效。</span><span class="sxs-lookup"><span data-stu-id="f16ac-131">However, this naming scheme has no effect on visibility, member access, inheritance, security, or binding.</span></span> <span data-ttu-id="f16ac-132">一个命名空间可以被划分在多个程序集中，而单个程序集可以包含来自多个命名空间的类型。</span><span class="sxs-lookup"><span data-stu-id="f16ac-132">A namespace can be partitioned across multiple assemblies and a single assembly can contain types from multiple namespaces.</span></span> <span data-ttu-id="f16ac-133">程序集为公共语言运行时中的版本控制、部署、安全性、加载和可见性提供外形结构。</span><span class="sxs-lookup"><span data-stu-id="f16ac-133">The assembly provides the formal structure for versioning, deployment, security, loading, and visibility in the common language runtime.</span></span>  
  
 <span data-ttu-id="f16ac-134">有关命名空间和类型名的更多信息，请参阅[通用类型系统](base-types/common-type-system.md)。</span><span class="sxs-lookup"><span data-stu-id="f16ac-134">For more information on namespaces and type names, see [Common Type System](base-types/common-type-system.md).</span></span>
